OSI model

layer 7

application

  • the user interface responsible for displaying received information to the user

layer 6

presentation

  • data translator or syntax layer
  • delivery and formatting of information to the application layer
  • MIME, XDR

layer 5

session

  • protocol mechanisms for opening, closing and managing sessions between processes
  • SOCKS, NetBIOS, etc

layer 4

transport

  • firewall - port filtering - TCP/UDP
  • some firewalls can filter through layer 7
  • can encrypt traffic I/O, and proxy traffic

overlap

  • most firewalls can also be routers
  • sits on ingress/egress of network

layer 3

network

  • router - routes traffic between IP subnets
  • routers inside of switches sometimes called layer 3 switches
  • often connects different network types - LAN, WAN, copper, fiber
  • WAP (wireless access point) is not a wireless router; it is a layer 3 router (wired interface) and a layer 2 bridge (wireless interface)

overlap

  • multilayer switch that includes routing functionality
  • layer 3: router
  • layer 2: switch

layer 2

data link

  • switch - ASIC (application specific integrated circuit) w/ bridging done in hardware
  • forwards traffic based on data link address
  • many ports and features, including PoE (power over ethernet)

layer 1

physical

  • hub/repeater - traffic enters one port, repeated to every other port
  • half-duplex: can either send or receive, but not both at same time
  • less efficient as speed increase - two flavors are 10megabit / 100megabit

network intrusion

NIDS vs NIPS

  • intrusions- exploits against OS, application, etc -- ex. buffer overflows, cross-site scripting
  • NIDS - network-based intrusion detection system
    can see intrusion but not prevent
  • NIPS - network-based prevention system
    can detect as well as stop the intrusion

identification technologies

  • signature-based- looks for a specific type of data -- aperfect match in the traffic
  • anomaly-based - establishes a baseline, then identify a deviation
  • behavior-based - observe and report
  • heuristics - artificial intelligence used to identify - not a particular signature but smart targeting tech

IDS implementations

  • software - ex. snort, load it and go -- liveCDs as well
  • hardware - specialized high-speed w/ enterprise features -- high availability, external logging
  • host-based - started as separate app, now integrated into many "endpoint" products (anti-virus, anti-spyware, etc) -- generally signature-based
    but can also function based on behavior (identifying user behavior)

content filtering

  • control traffic based on data within content
  • data in the packets -- specifically in the application section of the packet
  • ex: corporate control of sensitive materials through outbound and inbound data
  • ex: control of inappropriate content (NSFW, parental controls)
  • protection against "evil" -- anti-virus, anti-malware

email filtering

  • examines inbound and outbound email traffic
  • most common is spam filter
  • control of phishing attempts
  • anti-virus -- scan and block bad attachments

url filtering

  • allow or restrict based on URL (or URI -- uniform resource identifier)
  • commonly allow list / block list
  • managed by category -- auction, hacking, malware, travel, recreation, etc
  • downside: often unable to identify through encryption (advanced SSL encryption for instance)
  • some newer filters can decrypt SSL
